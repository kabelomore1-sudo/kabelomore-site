# kabelomore-site

The personal-brand and consulting site for **Kabelo More — AI Visibility Consulting**.

Built with Next.js 15 (App Router), Tailwind CSS, and TypeScript. Deployed on Vercel. Stripe-inspired typography and structure. AEO-first — the site is intentionally optimised for the AI engines we sell visibility to.

**Live domains:**
- `kabelomore.com` — primary, global
- `kabelomore.co.za` — secondary, redirects to primary

---

## Quick start

```bash
# Install
npm install

# Dev server (http://localhost:3000)
npm run dev

# Production build
npm run build && npm run start

# Type-check
npm run typecheck
```

Node 20+ required.

## Deploying to Vercel

### 1. Push to GitHub

```bash
cd C:/kabelomore-site
git init
git add .
git commit -m "Initial site — phase 1"
git branch -M main
git remote add origin https://github.com/kabelomore1-sudo/kabelomore-site.git
git push -u origin main
```

### 2. Connect to Vercel

1. Go to https://vercel.com/new
2. Click **Import Git Repository**
3. Select `kabelomore1-sudo/kabelomore-site`
4. Framework preset: **Next.js** (auto-detected)
5. Click **Deploy** — first deploy takes ~2 minutes

You'll get a temporary URL like `kabelomore-site-xxxx.vercel.app`. Test it works end-to-end before moving to step 3.

### 3. Add custom domains

In your Vercel project → **Settings** → **Domains**, add both:

- `kabelomore.com` (set as **Primary**)
- `www.kabelomore.com` (Vercel auto-redirects to apex)
- `kabelomore.co.za` (set as **Redirect** → `kabelomore.com`)
- `www.kabelomore.co.za` (auto-redirect to apex co.za, which then 301s to .com)

Vercel will show you the DNS records you need to add at your domain registrar.

### 4. DNS records to add at your registrar

**At Porkbun (kabelomore.com):**

| Type | Host | Value | TTL |
|---|---|---|---|
| A | @ | 76.76.21.21 | 300 |
| CNAME | www | cname.vercel-dns.com. | 300 |

**At Hetzner / domains.co.za (kabelomore.co.za):**

| Type | Host | Value | TTL |
|---|---|---|---|
| A | @ | 76.76.21.21 | 300 |
| CNAME | www | cname.vercel-dns.com. | 300 |

DNS propagation usually takes 10-60 minutes. Once propagated, Vercel auto-provisions HTTPS via Let's Encrypt.

### 5. Verify after deployment

After domains are live, sanity check:

- ✅ `https://kabelomore.com/` loads
- ✅ `https://kabelomore.co.za/` redirects to `kabelomore.com`
- ✅ `https://kabelomore.com/sitemap.xml` lists all pages
- ✅ `https://kabelomore.com/robots.txt` shows the AI crawler allowlist
- ✅ `https://kabelomore.com/llms.txt` returns plain text describing the business
- ✅ Free scan form at `/scan` submits without errors
- ✅ View source on `/` — JSON-LD `Organization` and `FAQPage` schemas visible
- ✅ Lighthouse score: aim for 95+ on Performance, 100 on Accessibility/SEO/Best Practices

### 6. Submit to search consoles

- **Google Search Console** — https://search.google.com/search-console — add `kabelomore.com`, submit sitemap
- **Bing Webmaster Tools** — https://www.bing.com/webmasters — add `kabelomore.com`, submit sitemap. Critical because Microsoft Copilot uses Bing's index.

## Optional: enable email notifications for scan submissions

By default, scan form submissions are logged to Vercel logs (View deployment → Logs tab). To also receive an email, set up Resend:

1. Sign up at https://resend.com (free tier: 100 emails/day)
2. Verify your sending domain — add the DNS records Resend provides at Porkbun
3. Create an API key at https://resend.com/api-keys
4. In Vercel project → **Settings** → **Environment Variables**, add:
   - `RESEND_API_KEY` = the key from step 3
   - `SCAN_INBOX_EMAIL` = `kabelo@kabelomore.com`
   - `SCAN_FROM_EMAIL` = `scan@kabelomore.com` (must be on a domain you verified in step 2)
5. Redeploy

Now every form submission also lands in your inbox.

## Adding new content

### A new case study

1. Create `app/(marketing)/case-studies/<slug>/page.tsx` (copy the OMS file as a template)
2. Add the entry to the array in `app/(marketing)/case-studies/page.tsx`
3. Add a sitemap entry in `app/sitemap.ts`
4. Push to main → Vercel auto-deploys

### A new blog post

The blog placeholder ships first. To enable real posts:

1. Install MDX: `npm install @next/mdx @mdx-js/loader @mdx-js/react`
2. Add MDX config to `next.config.ts` (see [Next.js docs](https://nextjs.org/docs/app/building-your-application/configuring/mdx))
3. Create `app/(marketing)/blog/[slug]/page.tsx` to render MDX from `content/blog/`
4. Update `app/(marketing)/blog/page.tsx` to list real posts

### Updating pricing

Open `lib/site.ts`. Change the `tiers` array. Save. Push. Done — pricing updates everywhere on the site at once.

## Stack reference

| Concern | Choice | Why |
|---|---|---|
| Framework | Next.js 15 + App Router | Vercel-native, server-rendered, RSC by default, great DX |
| Language | TypeScript strict | Catch bugs at write-time |
| Styling | Tailwind CSS 3.4 + custom tokens | Stripe-quality typography without dependency soup |
| Components | Local (shadcn-style) | No library lock-in, every component lives in this repo |
| Fonts | Geist Sans + Geist Mono | Vercel-shipped, free, closest free alternative to Söhne |
| Icons | Lucide React | Clean, consistent, lightweight |
| Forms | Native HTML + Zod validation | No form library overhead for 6 fields |
| Email | Resend (optional) | Best deliverability for transactional, generous free tier |
| Analytics | Vercel Analytics (recommended) | Free, privacy-friendly, no GDPR cookie banner needed |
